# 姿态估计模型研究报告 - MediaPipe 替代方案

> 研究日期: 2026-07-11
> 目标: 为瑜伽体式分析应用评估姿态估计模型替代方案

## 1. 当前方案: MediaPipe Pose

### 1.1 模型概述
- **关键点数量**: 33 个 (包含面部、手部、脚部)
- **运行时**: MediaPipe Runtime (WASM + GPU加速)
- **输入分辨率**: 256x256
- **模型大小**: 约 14MB (Heavy), 约 10MB (Full), 约 4MB (Lite)
- **推理速度**: 
  - MacBook Pro 15" 2019: 92-160 FPS (Heavy-Full-Lite)
  - iPhone 12: 81-140 FPS
  - Pixel 5: 38-98 FPS
  - Desktop (i9-10900K + GTX 1070): 32-98 FPS

### 1.2 精度指标 (COCO Keypoints)
| 数据集 | BlazePose.Heavy | BlazePose.Full | BlazePose.Lite |
|--------|-----------------|----------------|----------------|
| Yoga mAP | 68.1 | 62.6 | 45.0 |
| Yoga PCK@0.2 | 96.4 | 95.5 | 90.2 |
| Dance mAP | 73.0 | 67.4 | 53.6 |
| Dance PCK@0.2 | 97.2 | 96.3 | 92.5 |
| HIIT mAP | 74.0 | 68.0 | 53.8 |
| HIIT PCK@0.2 | 97.5 | 95.7 | 93.5 |

### 1.4 优势
- 高精度 (尤其 Heavy 变体)
- 完整的 33 个关键点 (面部、手部、脚部)
- 良好的浏览器兼容性 (WASM + WebGL)
- 实时性能优异
- 支持分割掩码
- Google 维护，社区活跃

### 1.5 劣势
- 模型较大 (Heavy: 34.9MB)
- 仅支持单人检测
- 依赖 Google 生态

## 2. 替代方案: MoveNet

### 2.1 模型变体

#### SinglePose Lightning
- **定位**: 最快的单人姿态检测器
- **关键点数量**: 17 个 (COCO 格式)
- **模型大小**: 约 4MB
- **推理速度**:
  - MacBook Pro 15" 2019: 104 FPS (WebGL), 42 FPS (WASM+SIMD)
  - iPhone 12: 51 FPS
  - Pixel 5: 34 FPS
  - Desktop (i9-10900K + GTX 1070): 87 FPS (WebGL), 71 FPS (WASM+SIMD)

#### SinglePose Thunder
- **定位**: 更高精度的单人检测器
- **关键点数量**: 17 个 (COCO 格式)
- **模型大小**: 约 10MB
- **推理速度**:
  - MacBook Pro 15" 2019: 77 FPS (WebGL), 21 FPS (WASM+SIMD)
  - iPhone 12: 43 FPS
  - Pixel 5: 12 FPS
  - Desktop (i9-10900K + GTX 1070): 82 FPS (WebGL), 30 FPS (WASM+SIMD)

#### MultiPose Lightning
- **定位**: 多人姿态检测 (最多 6 人)
- **关键点数量**: 17 个 (COCO 格式)
- **模型大小**: 约 4MB
- **推理速度**:
  - MacBook Pro 15" 2019: 54 FPS (WebGL)
  - iPhone 12: 24 FPS
  - Pixel 5: 8 FPS
  - Desktop (i9-10900K + GTX 1070): 62 FPS (WebGL)

### 2.2 关键点拓扑 (COCO)
```
0: nose
1: left_eye
2: right_eye
3: left_ear
4: right_ear
5: left_shoulder
6: right_shoulder
7: left_elbow
8: right_elbow
9: left_wrist
10: right_wrist
11: left_hip
12: right_hip
13: left_knee
14: right_knee
15: left_ankle
16: right_ankle
```

### 2.3 优势
- 极快的推理速度 (Lightning 变体)
- 模型轻量 (4-10MB)
- 支持多人检测 (MultiPose 变体)
- 良好的浏览器兼容性
- TensorFlow.js 官方维护
- 实时性能优异

### 2.4 劣势
- 仅 17 个关键点 (缺少面部、手部、脚部细节)
- 精度略低于 BlazePose Heavy
- 无分割掩码支持
- 无 3D 关键点输出

### 2.5 安装方式
```javascript
// NPM 安装
npm install @tensorflow-models/pose-detection @tensorflow/tfjs-core @tensorflow/tfjs-converter @tensorflow/tfjs-backend-webgl

// 使用
import * as poseDetection from '@tensorflow-models/pose-detection';
import '@tensorflow/tfjs-backend-webgl';

const detector = await poseDetection.createDetector(
  poseDetection.SupportedModels.MoveNet,
  {
    modelType: poseDetection.movenet.modelType.SINGLEPOSE_LIGHTNING,
    enableSmoothing: true
  }
);

const poses = await detector.estimatePoses(image);
```

## 3. 替代方案: BlazePose (TensorFlow.js Runtime)

### 3.1 模型变体
- **Lite**: 最小模型，精度较低，内存占用最小
- **Full**: 平衡性能和精度
- **Heavy**: 最大模型，最高精度

### 3.2 性能数据
| 设备 | Lite (FPS) | Full (FPS) | Heavy (FPS) |
|------|------------|------------|-------------|
| MacBook Pro 15" 2019 | 48 | 53 | 28 |
| iPhone 12 | 34 | 30 | N/A |
| Pixel 5 | 12 | 11 | 5 |
| Desktop (i9-10900K + GTX 1070) | 44 | 40 | 30 |

### 3.3 Bundle Size
| 组件 | 大小 (gzipped + minified) | 加载时间 (100Mbps) |
|------|---------------------------|---------------------|
| **MediaPipe Runtime** | | |
| Initial Page Load | 22.1KB | 0.04s |
| Runtime | 1.57MB | - |
| Lite model | 10.6MB | 1.91s |
| Full model | 14MB | 1.91s |
| Heavy model | 34.9MB | 4.82s |
| **TensorFlow.js Runtime** | | |
| Initial Page Load | 162.6KB | 0.07s |
| Lite model | 10.41MB | 1.91s |
| Full model | 13.8MB | 1.91s |
| Heavy model | 34.7MB | 4.82s |

### 3.4 优势
- 与 MediaPipe Pose 相同的精度指标
- 完整的 33 个关键点
- 支持分割掩码
- TensorFlow.js 生态集成良好

### 3.5 劣势
- 推理速度低于 MediaPipe Runtime
- 仅支持单人检测
- 模型较大 (Heavy: 34.7MB)

### 3.6 安装方式
```javascript
// NPM 安装
npm install @tensorflow-models/pose-detection @tensorflow/tfjs-core @tensorflow/tfjs-converter @tensorflow/tfjs-backend-webgl

// 使用
import * as poseDetection from '@tensorflow-models/pose-detection';
import '@tensorflow/tfjs-backend-webgl';

const detector = await poseDetection.createDetector(
  poseDetection.SupportedModels.BlazePose,
  {
    runtime: 'tfjs',
    enableSmoothing: true,
    modelType: 'full'
  }
);

const poses = await detector.estimatePoses(image, { flipHorizontal: true });
```

## 4. 替代方案: PoseNet

### 4.1 模型架构
- **MobileNetV1**: 较小、较快、精度较低
- **ResNet50**: 较大、较慢、精度较高

### 4.2 配置选项
```javascript
// MobileNet 配置
const config = {
  architecture: 'MobileNetV1',
  outputStride: 16,
  inputResolution: { width: 640, height: 480 },
  multiplier: 0.75
};

// ResNet 配置
const config = {
  architecture: 'ResNet50',
  outputStride: 32,
  inputResolution: { width: 257, height: 200 },
  quantBytes: 2
};
```

### 4.3 模型大小
- **4 bytes (no quantization)**: ~90MB
- **2 bytes**: ~45MB
- **1 byte**: ~22MB

### 4.4 关键点拓扑
与 MoveNet 相同的 17 个 COCO 关键点

### 4.5 优势
- 支持多人检测 (Multi-pose)
- 可配置架构和精度
- TensorFlow.js 官方维护
- 良好的文档和示例

### 4.6 劣势
- 推理速度较慢
- 模型较大
- 仅 17 个关键点
- 已被 MoveNet 取代 (官方推荐)

### 4.7 安装方式
```javascript
import * as poseDetection from '@tensorflow-models/pose-detection';
import '@tensorflow/tfjs-backend-webgl';

const detector = await poseDetection.createDetector(
  poseDetection.SupportedModels.PoseNet,
  {
    architecture: 'MobileNetV1',
    outputStride: 16,
    inputResolution: { width: 640, height: 480 }
  }
);

// 多人检测
const poses = await detector.estimatePoses(image, {
  maxPoses: 5,
  scoreThreshold: 0.5
});
```

## 6. 瑜伽/健身姿态检测适用性分析

### 6.1 瑜伽体式需求
- **精确的手部关键点**: 手印式 (Anjali Mudra)、树式 (Vrksasana) 等
- **精确的脚部关键点**: 战士式 (Virabhadrasana)、三角式 (Trikonasana) 等
- **脊柱对齐检测**: 山式 (Tadasana)、下犬式 (Adho Mukha Svanasana) 等
- **平衡检测**: 舞王式 (Natarajasana)、鹰式 (Garudasana) 等

### 6.2 模型推荐

#### 场景 1: 高精度瑜伽分析 (推荐)
- **模型**: MediaPipe Pose (Heavy 或 Full)
- **理由**: 
  - 33 个关键点覆盖面部、手部、脚部
  - 最高精度 (Yoga mAP: 62.6-68.1)
  - 实时性能满足需求
  - 支持 3D 关键点 (体式深度分析)

#### 场景 2: 实时多人群体课程
- **模型**: MoveNet MultiPose Lightning
- **理由**:
  - 支持最多 6 人同时检测
  - 极快的推理速度 (54-62 FPS)
  - 轻量模型 (~4MB)
- **限制**: 仅 17 个关键点，缺少手部/脚部细节

#### 场景 3: 轻量级单人检测
- **模型**: MoveNet SinglePose Lightning
- **理由**:
  - 最快的推理速度 (87-104 FPS)
  - 最小的模型大小 (~4MB)
  - 适合低性能设备
- **限制**: 仅 17 个关键点

#### 场景 4: 平衡精度和性能
- **模型**: BlazePose Full (TensorFlow.js Runtime)
- **理由**:
  - 33 个关键点
  - 中等模型大小 (14MB)
  - 良好的推理速度 (30-53 FPS)
- **限制**: 无分割掩码，单人检测

### 6.3 关键点覆盖对比

|----------|----------------|---------|---------|

| 手臂 | ✅ (6点) | ✅ (6点) | ✅ (6点) |
| 手部 | ✅ (12点) | ❌ | ❌ |

## 9. 结论

### 9.1 最佳选择
**MediaPipe Pose (Heavy 或 Full)** 仍然是瑜伽/健身应用的最佳选择，原因：
1. 最高精度 (Yoga mAP: 62.6-68.1)
2. 完整的 33 个关键点 (面部、手部、脚部)
3. 实时性能优异 (92-160 FPS)
4. 支持 3D 关键点和分割掩码
5. Google 官方维护，生态完善

### 9.2 备选方案
**MoveNet MultiPose Lightning** 适合多人场景，但缺少手部/脚部关键点。

### 9.3 不推荐
**PoseNet** 已被 MoveNet 取代，不建议新项目使用。

### 9.4 下一步行动
1. 保持 MediaPipe Pose 作为主要模型
2. 评估 MoveNet 作为备选 (快速检测/多人场景)
3. 优化当前实现的性能
4. 考虑 SMPL 模型集成 (体式深度分析)
