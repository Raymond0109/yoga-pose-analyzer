# 高精度姿态估计模型深度研究报告

> 研究日期: 2026-07-11
> 目标: 评估 ViTPose、DWPose、MMPose、RTMPose 在瑜伽体式分析中的适用性
> 关联研究简报: brief.md
> 前序报告: F1.md (MediaPipe/MoveNet/BlazePose/PoseNet 基线评估)

---

## 0. 执行摘要

本报告聚焦四种 Transformer/MMPose 生态的高精度姿态估计模型，评估其替代或补充 MediaPipe 用于瑜伽体式分析的可行性。

### 关键发现

| 维度 | ViTPose | DWPose | MMPose (框架) | RTMPose |
|------|---------|--------|--------------|---------|
| **COCO AP** | 73.8–81.1 | 58.5–72.2 (WholeBody) | 取决于选用模型 | 68.5–78.8 |
| **关键点数** | 17 (COCO) | 133 (WholeBody) | 17–133+ | 17/26/133 |
| **推理速度 (CPU)** | ~10–30 FPS | ~15–40 FPS | 取决于模型 | **50–300+ FPS** |
| **模型大小** | 86–632M params | 3.3–27.7M params | 取决于模型 | **3.3–49.4M params** |
| **ONNX 导出** | 需手动转换 | ✅ 已提供 | ✅ MMDeploy 支持 | ✅ 已提供 |
| **浏览器 WASM** | 困难 | 理论可行 | 需适配 | **最佳候选** |

### 瑜伽场景推荐优先级
1. **RTMPose-m/l (17 keypoint)** — 实时性最佳，精度高，ONNX 已就绪
2. **DWPose-l** — 133 关键点全身体估计，对手部/面部细节瑜伽动作更精确
3. **ViTPose-B/L** — 最高学术精度，但部署复杂度高
4. **MMPose 框架** — 作为训练/部署工具链，不是单一模型

## 1. ViTPose / ViTPose++

### 1.1 模型概述
- **论文**: NeurIPS 2022 (ViTPose) + TPAMI 2023 (ViTPose++)
- **架构**: Vision Transformer (ViT) 作为 backbone + 简单 heatmap decoder
- **关键创新**: 证明朴素 ViT 架构在姿态估计任务上的 scaling law — 模型越大精度越高
- **GitHub**: [ViTAE-Transformer/ViTPose](https://github.com/ViTAE-Transformer/ViTPose) (2.1k stars)
- **依赖**: PyTorch + mmcv + mmpose (非独立可部署)

### 1.2 COCO Keypoint 精度 (单任务训练)

| 模型 | 预训练 | 输入分辨率 | COCO AP | AR | 参数量 (估) |
|------|--------|-----------|---------|-----|------------|
| ViTPose-S | MAE | 256x192 | **73.8** | 79.2 | ~22M |
| ViTPose-B | MAE | 256x192 | **75.8** | 81.1 | ~86M |
| ViTPose-L | MAE | 256x192 | **78.3** | 83.5 | ~307M |
| ViTPose-H | MAE | 256x192 | **79.1** | 84.1 | ~632M |
| ViTPose-G | MAE | 576x432 | **81.1** | 85.6 | ~1B+ |

### 1.3 COCO Keypoint 精度 (多任务训练)

| 模型 | 训练数据 | COCO AP |
|------|---------|---------|
| ViTPose-B | COCO+AIC+MPII | **77.1** |
| ViTPose-L | COCO+AIC+MPII | **78.7** |
| ViTPose-H | COCO+AIC+MPII | **79.5** |
| ViTPose-G | COCO+AIC+MPII | **81.0** |

### 1.4 ViTPose++ 精度 (跨域通用)

| 模型 | 训练数据 | COCO AP | OCHuman AP |
|------|---------|---------|-----------|
| ViTPose++-S | COCO+AIC+MPII+AP10K+APT36K+WholeBody | 75.8 | 78.4 |
| ViTPose++-B | 同上 | 77.0 | 82.6 |
| ViTPose++-L | 同上 | 78.6 | 85.7 |
| ViTPose++-H | 同上 | 79.4 | 85.7 |

### 1.5 关键点定义
- **17 个 COCO 关键点** (标准人体)
- ViTPose++ 支持 WholeBody (133 点)、手部、面部、动物
- WholeBody AP: ViTPose++-H = 61.2

### 1.6 推理速度
- **无官方浏览器/移动端 benchmark**
- GPU (RTX 3090): ViTPose-B ~45 FPS, ViTPose-L ~15 FPS, ViTPose-H ~7 FPS
- **瓶颈**: ViT 的 self-attention 计算复杂度 O(n²)，输入分辨率越大越慢

### 1.7 浏览器/WASM 部署评估

| 维度 | 评估 |
|------|------|
| ONNX 导出 | 需手动转换 (非官方支持) |
| WASM 推理 | **极困难** — ViT 计算量大，WASM 无 SIMD 优化 Transformer 效率极低 |
| WebGPU | 理论可行但实现复杂，ViTPose 无官方 WebGPU 支持 |
| ONNX Runtime Web | 可尝试但性能堪忧 (ViTPose-B ~86M params 在浏览器中极慢) |
| 模型大小 | ViTPose-S: ~88MB, ViTPose-B: ~344MB, ViTPose-L: ~1.2GB |

### 1.8 瑜伽适用性评估

| 优势 | 劣势 |
|------|------|
| COCO AP 最高 (75.8–81.1) | 模型太大，无法实时 |
| Transformer 全局感受野适合极端姿态 | 无官方浏览器/ONNX 导出 |
| ViTPose++ 支持跨域泛化 | 推理速度慢 (GPU <50 FPS) |
| 学术 SOTA | 部署工程量巨大 |

**结论**: ViTPose 是学术标杆，但不适合 Electron 实时应用。除非后续 ViT 蒸馏出轻量版本，否则不推荐直接集成。

## 2. DWPose (Dual-Wholistic Pose)

### 2.1 模型概述
- **论文**: "Effective Whole-body Pose Estimation with Two-stages Distillation" (ICCV 2023, CV4Metaverse Workshop)
- **架构**: 基于 RTMPose 架构 + 两阶段知识蒸馏 (teacher-student)
- **关键创新**: 使用大模型 (ViTPose-H/G) 作为 teacher，蒸馏出高效 WholeBody 模型
- **GitHub**: [IDEA-Research/DWPose](https://github.com/IDEA-Research/DWPose) (2.8k stars)
- **来源**: IDEA Research (DINO/DINOv2 团队)
- **依赖**: 基于 MMPose 框架

### 2.2 COCO-WholeBody 精度

| 模型 | 输入分辨率 | FLOPs (G) | Body AP | Foot AP | Face AP | Hand AP | Whole AP |
|------|-----------|-----------|---------|---------|---------|---------|----------|
| DWPose-t | 256x192 | 0.5 | 0.585 | 0.465 | 0.735 | 0.357 | **0.485** |
| DWPose-s | 256x192 | 0.9 | 0.633 | 0.533 | 0.776 | 0.427 | **0.538** |
| DWPose-m | 256x192 | 2.2 | 0.685 | 0.636 | 0.828 | 0.527 | **0.606** |
| DWPose-l | 256x192 | 4.5 | 0.704 | 0.662 | 0.843 | 0.566 | **0.631** |
| DWPose-l | 384x288 | 10.1 | 0.722 | 0.704 | 0.887 | 0.621 | **0.665** |

> 注: 使用 human AP=56.4 的 detector

### 2.3 关键点定义
- **133 个关键点 (COCO-WholeBody)**:
  - Body: 17 点 (标准 COCO)
  - Foot: 6 点 (每脚 3 点)
  - Face: 68 点 (面部网格)
  - Hand: 42 点 (每手 21 点)
- **对瑜伽特别有价值**: 手指关节 (手印/Mudra)、脚趾 (平衡体式)、面部表情

### 2.4 推理速度
- DWPose-l (256x192): ONNX Runtime ~23ms (i7-11700), TensorRT ~5.7ms (GTX 1660Ti)
- DWPose-m (256x192): ONNX Runtime ~13.5ms, TensorRT ~4.0ms
- DWPose-s (256x192): ONNX Runtime ~4.5ms (估计), TensorRT ~1.4ms
- DWPose-t (256x192): ONNX Runtime ~3.2ms (估计), TensorRT ~1.1ms

### 2.5 ONNX 导出与部署
- **官方提供 ONNX 模型**: [HuggingFace yzd-v/DWPose](https://huggingface.co/yzd-v/DWPose/tree/main)
- **ONNX 模型列表**:
  - `dw-ll_ucoco_384.onnx` (大模型, 384x288)
  - `dw-mm_ucoco_256x192.onnx` (中模型)
  - `dw-sm_ucoco_256x192.onnx` (小模型)
  - `dw-ts_ucoco_256x192.onnx` (最小模型)
- **ONNX Runtime Web 可行性**: ONNX 格式天然支持 ONNX Runtime Web (WASM/WebGPU)

### 2.6 浏览器/WASM 部署评估

| 维度 | 评估 |
|------|------|
| ONNX 导出 | ✅ 官方已提供 |
| WASM 推理 | ✅ 理论可行 (ONNX Runtime Web 支持) |
| 模型大小 | DWPose-t: ~7MB, DWPose-s: ~11MB, DWPose-m: ~27MB, DWPose-l: ~55MB |
| WebGPU 加速 | ✅ ONNX Runtime Web 支持 WebGPU EP |
| 预估浏览器 FPS | DWPose-t: ~15-25 FPS, DWPose-s: ~10-20 FPS, DWPose-m: ~5-10 FPS |

### 2.7 瑜伽适用性评估

| 优势 | 劣势 |
|------|------|
| **133 关键点**: 手指/脚趾/面部全覆盖 | 模型仍偏大 (l: 55MB) |
| Body/Foot/Face/Hand 分区 AP | 浏览器推理速度可能不足 |
| 官方 ONNX 模型可用 | 依赖 MMPose 框架 |
| 知识蒸馏效果好 | 精度 (WholeBody AP 0.631) 低于 ViTPose |
| 适合 ControlNet 等下游任务 | 无官方 WASM demo |

**结论**: DWPose 是 **全身体关键点最佳选择**，133 点对手指/脚趾精度远超 MediaPipe。DWPose-t/s 可尝试 WASM 部署。建议作为 MediaPipe 的补充模型，用于需要手部/脚部精细检测的场景。

## 3. MMPose 框架

### 3.1 概述
- **定位**: OpenMMLab 姿态估计工具箱 (非单一模型)
- **GitHub**: [open-mmlab/mmpose](https://github.com/open-mmlab/mmpose) (7.7k stars, 1.5k forks)
- **最新版本**: v1.3.2 (2024-07-12)
- **核心价值**: 统一的训练/评估/部署流程，支持 30+ 算法
- **依赖**: PyTorch 1.8+, MMEngine, MMCV

### 3.2 支持的算法

| 算法 | 类型 | 关键点数 | 精度范围 (COCO AP) |
|------|------|---------|-------------------|
| HRNet | CNN | 17 | 74.4–75.1 |
| SimCC | CNN | 17 | 75.0–76.0 |
| **RTMPose** | CNN (实时) | 17/26/133 | 68.5–78.8 |
| ViTPose | ViT | 17/133 | 73.8–81.1 |
| Hourglass | CNN | 17 | 72.0–74.0 |
| SimpleBaseline | CNN | 17 | 70.0–72.0 |
| LiteHRNet | 轻量CNN | 17 | 68.0–71.0 |

### 3.3 部署支持

| 后端 | 支持状态 | 说明 |
|------|---------|------|
| ONNX Runtime | ✅ | 通过 MMDeploy |
| TensorRT | ✅ | FP16/INT8 |
| ncnn | ✅ | 移动端 |
| OpenVINO | ✅ | Intel 优化 |
| CoreML | ✅ | Apple 设备 |
| TorchScript | ✅ | PyTorch 原生 |
| **ONNX Runtime Web** | ⚠️ 间接 | 导出 ONNX 后可用 onnxruntime-web |

### 3.6 瑜伽适用性评估

| 优势 | 劣势 |
|------|------|
| 最完整的姿态估计生态 | 框架本身不是模型，需要选择具体算法 |

## 4. RTMPose (Real-Time MMPose)

- **GitHub**: MMPose 项目子目录 [projects/rtmpose](https://github.com/open-mmlab/mmpose/tree/main/projects/rtmpose)

## 6. 瑜伽极端体式分析

### 6.1 MediaPipe 失败场景分析

根据 F1.md 研究, MediaPipe 在以下瑜伽场景中表现不佳:

| 体式 | 问题 | 需要的关键点 |
|------|------|-------------|
| 下犬式 (Adho Mukha Svanasana) | 倒V形，头部低于心脏 | 脊柱对齐，肩膀角度 |
| 舞王式 (Natarajasana) | 单腿站立+后弯 | 脚趾平衡，脊柱弯曲度 |
| 手倒立 (Adho Mukha Vrksasana) | 完全倒置 | 全身关键点倒置检测 |
| 鸽子式 (Eka Pada Rajakapotasana) | 复杂髋部打开 | 髋部关键点，脊柱旋转 |
| 莲花式 (Padmasana) | 腿部交叉盘坐 | 脚部关键点精确位置 |
| 鹰式 (Garudasana) | 手臂/腿部缠绕 | 手指关节，膝盖角度 |

### 6.2 模型在极端体式中的预期表现

| 体式类型 | MediaPipe | RTMPose-17 | RTMPose-133 | DWPose-133 | ViTPose |
|----------|-----------|------------|-------------|------------|---------|
| 站立体式 | ★★★★ | ★★★★ | ★★★★★ | ★★★★★ | ★★★★★ |
| 坐立体式 | ★★★ | ★★★ | ★★★★ | ★★★★★ | ★★★★ |
| 平衡体式 | ★★ | ★★★ | ★★★★ | ★★★★ | ★★★★ |
| 后弯体式 | ★★ | ★★★ | ★★★★ | ★★★★ | ★★★★★ |
| 倒立体式 | ★ | ★★ | ★★★ | ★★★★ | ★★★★ |
| 手指/Mudra | ★★★★ | ★ | ★★★★ | ★★★★★ | ★★★★ |

### 6.3 为什么 ViTPose/RTMPose 对极端体式更好

1. **全局感受野 (ViT/CNN)**: Transformer 的 self-attention 可捕捉全身上下文关系，即使身体部分遮挡或极端角度
2. **COCO 数据集多样性**: COCO 包含各种角度/姿态的人体，模型天然对非标准姿态鲁棒
3. **Heatmap/SimCC 回归**: 比 MediaPipe 的轻量级 BlazePose 架构有更强的特征表达能力
4. **133 关键点 (DWPose)**: 覆盖手指/脚趾/面部，在精细动作中更准确

## 8. 模型选择决策树

```
需要实时 (< 30 FPS)?
├── 是 → 需要面部/手指细节?
│   ├── 是 → DWPose-t (7MB, 48.5 WholeBody AP)
│   └── 否 → RTMPose-m (27MB, 75.8 AP) ← 推荐
└── 否 → 需要最高精度?
    ├── 是 → ViTPose-L (1.2GB, 78.3 AP) — 仅服务端
    └── 否 → RTMPose-l-384 (55MB, 78.3 AP)
```

### 具体推荐

#### 主力模型: RTMPose-m (27MB, 75.8 AP)
- **理由**: 精度比 MediaPipe 高 ~8%，速度相当，模型更小
- **部署**: ONNX → onnxruntime-web (WASM/WebGPU)
- **关键点**: 17 个 COCO 点，可扩展到 26 点 (Halpe)

#### 补充模型: DWPose-t (7MB, 48.5 WholeBody AP)
- **理由**: 133 点全身体估计，手指/脚趾/面部全覆盖
- **使用场景**: 需要精细手部动作分析时启用
- **部署**: 同样 ONNX → onnxruntime-web

#### 未来方向: 自定义 Yoga-RTMPose
- **方法**: 使用 RTMPose 架构 + 瑜伽数据集 fine-tune
- **优势**: 针对瑜伽体式优化的关键点权重
- **所需**: 瑜伽图像数据集 + 关键点标注

## 9. 风险与限制

### 9.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| WASM 推理速度不达标 | 流畅度差 | WebGPU 优先；降级到 RTMPose-t |
| ONNX 算子不兼容 | 模型加载失败 | 使用官方预编译 ONNX；避免自定义算子 |
| 浏览器兼容性 | 部分功能不可用 | Electron 环境可控；检测 WebGPU 支持 |
| 模型大小影响启动 | 首次加载慢 | 模型懒加载 + 缓存策略 |

### 9.2 精度限制

| 限制 | 说明 |
|------|------|
| 无瑜伽专用训练数据 | COCO/WholeBody 数据集以日常动作为主 |
| 极端遮挡仍可能失败 | 手倒立等完全遮挡场景 |
| 3D 信息缺失 | 2D 关键点无法完全捕捉脊柱弯曲度 |
| 动态模糊 | 快速动作可能降低精度 |
